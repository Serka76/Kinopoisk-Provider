# Changelog

Формат: `[версия] — дата` + что изменилось. Версия видна в:
- логах при старте контейнера (`docker compose logs -f` / вкладка «Журнал» в Container Manager)
- ответе `GET /healthz`
- названии провайдера в Plex (Settings → Metadata Agents)

После обновления файлов **обязательно пересобирай контейнер**
(«Действие» → «Собрать» → «Пуск») — иначе новая версия не подхватится.
Если версия не поменялась после сборки — удали образ в разделе «Образ»
(Image) и собери заново с нуля, Docker иногда слишком агрессивно кэширует.

## [0.15.0] — 2026-07-19

- **Найдена вероятная причина "фото не появились" при tmdb_photos_found>0**:
  поиск через прокси работал, но сам Plex скачивает картинки НАПРЯМУЮ,
  в обход нашего Hysteria2-прокси — и упирается в ту же блокировку на

  через настроенный прокси и отдаёт её от своего имени

  вместо прямой ссылки на TMDB, если задан новый `PUBLIC_BASE_URL`
- **Требуется новая переменная `PUBLIC_BASE_URL`** — тот же адрес, что
  вписан в Plex как URL провайдера. Без неё поведение как раньше
  (прямая ссылка на TMDB, которую Plex не сможет скачать)
- **НЕПОДТВЕРЖДЕНО**: гипотеза логичная и хорошо объясняет симптом
  (поиск нашёл 72/33 фото, но карточка совсем без фото), но не

## [0.14.0] — 2026-07-19

- **Автопарсинг ссылки Hysteria2**: больше не нужно вручную разбирать
  `hysteria2://...` на сервер/пароль/SNI — теперь достаточно вставить
  ссылку целиком в переменную `HYSTERIA2_URL`, новый sidecar-контейнер
  (`hysteria-proxy/`, свой Dockerfile + `entrypoint.sh`) сам собирает
  конфиг при старте. Скорее всего, причина "фото так и не появились"
  была именно в ручном заполнении старого YAML-шаблона — этот способ
  проще и меньше шансов ошибиться
- Убран `hysteria-client.yaml.example` (больше не нужен)
- `docker-compose.yml` теперь содержит реальные секреты (ключи + ссылку
  на прокси) и добавлен в `.gitignore`; для публикации проекта — новый
  `docker-compose.yml.example` с плейсхолдерами
- **НЕПОДТВЕРЖДЕНО**: сам парсинг ссылки в `entrypoint.sh` не проверен
  вживую (нет тестовой ссылки под рукой) — если что-то не соберётся,
  смотри лог контейнера `hysteria-proxy`, там печатается разобранный
  результат (без пароля)

## [0.12.6] — 2026-07-19

- **Найдена настоящая причина `tmdb_photos_found=0`**: не блокировка
  по географии, а `OSError: [Errno 99] Cannot assign requested
  address` — у контейнера на Synology нет рабочего исходящего IPv6, а
  у TMDB (в отличие от poiskkino.dev) есть AAAA-запись, и Python
  пытался подключиться именно по ней. Форсирую IPv4 через
  `httpx.AsyncHTTPTransport(local_address="0.0.0.0")` — стандартный
  обходной приём для httpx в такой ситуации
- Диагностировано через прямое подключение к контейнеру по SSH и
  ручной `python3 -c "urllib.request.urlopen(...)"` — спасибо за
  терпение с диагностикой через терминал
